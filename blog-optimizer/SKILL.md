# Blog Optimizer

## 概述

本技能旨在提升博客内容 Markdown 文件的质量。它通过增强图片描述（ALT 文本）、润色文字表达，并基于正文生成简洁的描述信息填充 frontmatter 的 description 字段，使内容更具可读性和信息量，同时严格保持原文的忠实度。

## 优化准则

在执行优化任务时，请遵循以下核心原则：

### 1. 图片描述强化
- **禁止修改路径**：绝对不要更改任何图片引用的 URL 或文件路径。
- **上下文推断**：阅读图片前后的文本内容，推断图片所传达的信息或场景。
- **生成描述**：在图片的 `[]` 括号内填入生成的描述信息。
  - *示例前*：`![](images/chart.png)`
  - *示例后*：`[显示 2023 年全球博客市场增长趋势的折线图](images/chart.png)`

### 2. 表达润色
- **完善句式**：修正语法错误，优化词汇选择，使表达更符合中文阅读习惯。
- **逻辑连贯**：增强段落间的过渡，确保博客转录带来的口语化内容转化为流畅的书面文字。
- **保持风格**：在不改变原意的前提下进行润色。

### 3. 内容保真
- **严禁扩展**：不要添加任何与原文无关的外部信息或个人见解。
- **完整性**：确保原始信息的关键点没有在优化过程中丢失。

### 4. 描述信息生成
- **基于正文**：从标题和正文内容中提炼关键信息生成描述，避免主观扩展。
- **长度克制**：保持简洁，适合作为文章摘要与索引说明。
- **更新字段**：只更新 frontmatter 中的 `description:` 值，不新增或改动其他字段。
- **格式示例**：
  - *示例前*：
    - `description:`
  - *示例后*：
    - `description: 生成的信息`

## 常用操作流程

1. **读取文件**：全面阅读待优化的博客 Markdown 文件。
2. **逐项处理**：
   - 定位所有图片标记，结合语境编写描述。
   - 对正文进行润色。
   - 基于文章内容生成描述并更新 frontmatter 的 `description` 字段。
3. **核对验证**：检查是否误改了路径、frontmatter 字段，或添加了无关内容。
4. **输出结果**：提供优化后的完整 Markdown 内容。
